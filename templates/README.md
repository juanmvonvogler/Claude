# templates/

Reusable starting points for creating new agents and skills. Copy, rename, and fill in — these keep new additions structurally consistent with the rest of the framework.

**What belongs here:** skeleton files with the standard sections and frontmatter expected of agents and skills, plus any conventions reviewers will look for.

**What does NOT belong here:** fully-written agents or skills (those live in their proper folders), and any example content that references specific users, projects, or private data.
