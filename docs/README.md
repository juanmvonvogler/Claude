# docs/

Documentation about the framework itself — architecture, conventions, and contribution guidelines. These files explain how the system fits together, not how to use any particular agent.

**What belongs here:** architecture diagrams, onboarding guides, conventions, and contribution notes.

**What does NOT belong here:** user-specific usage journals, personal notes, or any private context. If a document starts to read like "I did X on project Y", it belongs in the private context system instead.
