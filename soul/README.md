# soul/

The "why" layer — identity, principles, and operating philosophy that apply across every agent in the framework.

**What belongs here:** cross-cutting guidance an agent should embody regardless of its specific role — communication norms, decision heuristics, values that shape tone and trade-offs.

**What does NOT belong here:** role-specific instructions (those live in `agents/` or `skills/`), and any personal context — no names, projects, or private preferences. Keep everything generic enough to apply to any user of the framework.
